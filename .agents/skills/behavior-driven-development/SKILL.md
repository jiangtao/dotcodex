# Behavior-Driven Development

## Core Principles

1. Understand behavior before coding.
2. Write scenarios in Given/When/Then form.
3. Drive implementation through failing tests.

## Iron Law

No production code without a failing test first.

## Practice

- Store behavior in executable scenarios where possible.
- Keep one behavior per scenario.
- Verify RED before writing production code.
- Keep GREEN minimal.
- Refactor only after the scenario passes.
