# Build Like the Original iPhone Team

Use this as an escalation path inside `brainstorming` when the request calls for unconventional product thinking rather than incremental iteration.

## When To Load It

Load this skill when:

- industry conventions are the main constraint
- the user wants a category-defining approach
- standard option analysis keeps converging on obvious answers
- the problem needs first-principles thinking, not optimization

Do not load it for routine feature delivery or straightforward maintenance work.

## Operating Principles

1. Question every assumption from zero.
2. Force breakthrough options onto the table before accepting conventional limits.
3. Specify the experience in human terms before settling implementation details.
4. Let competing concepts fight until one wins clearly.
5. Protect the exploration from outside noise and cargo-cult patterns.
